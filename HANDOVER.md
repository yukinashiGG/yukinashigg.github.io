## ギャラリー自動ビルドシステム

### 仕組み
`build_gallery.py` が `_Source/` フォルダを走査し、以下を自動実行：
1. 各画像を **webp に圧縮**（長辺最大2000px、品質85%）
2. `サムネ.png` → `thumb.webp` にリネーム
3. `index.html` 内のギャラリーHTMLを**自動書き換え**

### 運用フロー（ユーザーが普段やること）
```
1. gallery/_Source/ にフォルダと画像を追加・編集
2. 「更新した」と言うだけ
3. → build_gallery.py 実行 → git add/commit/push → デプロイ完了
```

### フォルダ命名規則
```
[番号]_[カテゴリ]/[番号]_[表示名][ 説明テキスト]/
  例: 01_VTuberモデル用イラスト/01_不知火燈様[ VTuber向け キャラクターデザイン・イラスト制作]/
```
- **番号** → 表示順の制御
- **[ ]内** → カードのサブタイトルとして表示
- **サムネ.png** → 各フォルダに必須（カードのサムネイル画像）

### ビルドコマンド
```bash
cd c:\Users\shuta\c_local_date\ユキナシHP
python build_gallery.py; git add -A; git commit -m "Update gallery"; git push origin main
```
